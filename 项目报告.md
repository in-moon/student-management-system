## 一、需求分析

### 1.1 项目背景

随着高校扩招，学生人数持续增长，传统的手工或Excel管理模式已经无法满足教务管理的实际需求。各学院、辅导员和教务人员需要一套高效、可靠的信息化系统来完成学生的日常管理工作。本项目旨在开发一个基于 Web 的学生管理系统，以**解决实际的教学管理问题**。

### 1.2 功能需求

| 编号 | 功能模块 | 描述 |
|------|---------|------|
| F1 | 用户认证 | JWT 登录鉴权，BCrypt 密码加密 |
| F2 | 学生管理 | 学生信息增删查改、分页搜索、多条件筛选 |
| F3 | 班级管理 | 班级信息维护、辅导员分配、学生人数统计 |
| F4 | 课程管理 | 课程信息维护、学分设置、学期安排 |
| F5 | 选课管理 | 学生选课/退课、成绩录入、选课状态跟踪 |
| F6 | AI 智能助手 | 基于大语言模型的智能问答，支持流式对话 |
| F7 | 数据缓存 | Redis 缓存热点数据，提升查询性能 |

### 1.3 非功能需求

- **性能**：常用查询响应 < 200ms，数据库查询经索引优化
- **安全**：JWT 令牌认证，BCrypt 密码加密，前后端分离 CORS 控制
- **可维护性**：采用 RESTful API 设计，Swagger 接口文档自动生成
- **可用性**：AI 服务降级机制，API 不可用时自动切换为本地模拟响应

## 二、数据库结构设计

### 2.1 E-R 模型

系统包含 6 个实体及其关系：

```
┌──────────────┐       ┌──────────────┐       ┌──────────────┐
│     User     │       │    Class     │       │   Student    │
├──────────────┤       ├──────────────┤       ├──────────────┤
│ id (PK)      │       │ id (PK)      │       │ id (PK)      │
│ username     │       │ class_no     │       │ student_no   │
│ password     │       │ name         │       │ name         │
│ role         │       │ grade        │       │ gender       │
│ created_at   │       │ major        │       │ age          │
└──────────────┘       │ counselor    │       │ class_id(FK) │
       │               │ student_count│       │ major        │
       │ 1             └──────────────┘       │ enroll_date  │
       │                     1                │ phone        │
       │                     │                │ email        │
       │ 记录               N│ 属于           └──────────────┘
       ▼                     ▼                       │
┌──────────────┐       ┌──────────────┐              │ N
│AI_Conversation│      │   Student    │              │
└──────────────┘       └──────────────┘              │
                              │                      │
                              │ N                    │
                              │                      │
                              ▼                      ▼
                       ┌──────────────┐       ┌──────────────┐
                       │  Enrollment  │       │    Course    │
                       ├──────────────┤       ├──────────────┤
                       │ id (PK)      │  N:1  │ id (PK)      │
                       │ student_id(FK)───────│ course_no    │
                       │ course_id(FK)│       │ name         │
                       │ score        │       │ credit       │
                       │ status       │       │ teacher      │
                       │ enroll_date  │       │ semester     │
                       └──────────────┘       │ max_students │
                                              └──────────────┘
```

**实体关系描述**：
- User（用户）与 AI_Conversation（AI对话）：**1:N** — 一个用户可有多条对话记录
- Class（班级）与 Student（学生）：**1:N** — 一个班级包含多名学生
- Student（学生）与 Course（课程）：**N:M** — 通过 Enrollment（选课表）关联
- Enrollment 包含选课日期、成绩、状态等附加属性

### 2.2 关系模型

#### user 表（用户表）
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(64) | NOT NULL, UNIQUE | 用户名 |
| password | VARCHAR(255) | NOT NULL | BCrypt 加密密码 |
| role | VARCHAR(32) | DEFAULT 'ADMIN' | 角色 |
| created_at | DATETIME | DEFAULT NOW | 创建时间 |

#### class 表（班级表）
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 班级ID |
| class_no | VARCHAR(64) | NOT NULL, UNIQUE | 班级编号 |
| name | VARCHAR(100) | NOT NULL | 班级名称 |
| grade | VARCHAR(20) | — | 年级 |
| major | VARCHAR(100) | INDEX | 专业 |
| counselor | VARCHAR(100) | — | 辅导员 |
| student_count | INT | DEFAULT 0 | 学生人数 |

#### student 表（学生表）
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 学生ID |
| student_no | VARCHAR(64) | NOT NULL, UNIQUE | 学号 |
| name | VARCHAR(100) | NOT NULL, INDEX | 姓名 |
| gender | VARCHAR(10) | — | 性别 |
| age | INT | — | 年龄 |
| class_id | BIGINT | FK→class.id, INDEX | 班级外键 |
| major | VARCHAR(100) | INDEX | 专业 |
| enroll_date | DATE | INDEX | 入学日期 |
| phone | VARCHAR(50) | — | 电话 |
| email | VARCHAR(100) | — | 邮箱 |

#### course 表（课程表）
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 课程ID |
| course_no | VARCHAR(64) | NOT NULL, UNIQUE | 课程编号 |
| name | VARCHAR(200) | NOT NULL, INDEX | 课程名称 |
| credit | DECIMAL(3,1) | — | 学分 |
| teacher | VARCHAR(100) | INDEX | 授课教师 |
| semester | VARCHAR(50) | INDEX | 学期 |
| max_students | INT | — | 最大容量 |
| description | TEXT | — | 课程描述 |

#### enrollment 表（选课表 — 学生-课程多对多）
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 记录ID |
| student_id | BIGINT | FK→student.id, INDEX | 学生ID |
| course_id | BIGINT | FK→course.id, INDEX | 课程ID |
| enroll_date | DATE | — | 选课日期 |
| score | DECIMAL(5,2) | INDEX | 成绩 |
| status | VARCHAR(20) | INDEX | ENROLLED/COMPLETED/DROPPED |
| — | — | **UNIQUE(student_id, course_id)** | 防重复选课 |

#### ai_conversation 表（AI对话历史表）
| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 记录ID |
| user_id | BIGINT | FK→user.id, INDEX | 用户ID |
| role | VARCHAR(20) | NOT NULL | user/assistant |
| content | TEXT | NOT NULL | 对话内容 |
| model | VARCHAR(100) | — | AI模型名 |
| created_at | DATETIME | INDEX | 创建时间 |

### 2.3 数据库规范化

所有表满足 **3NF（第三范式）**：
- 每个字段原子不可分 (1NF)
- 非主键字段完全函数依赖于主键 (2NF)  
- 不存在传递函数依赖 (3NF)

## 三、功能模块详解

### 3.1 系统架构

```
┌──────────────────────────────────────────────────┐
│                   Frontend (Vue3)                 │
│  Element Plus UI  │  Vue Router  │  Axios HTTP    │
│               Port: 5173 (Dev)                    │
└─────────────────────┬────────────────────────────┘
                      │ REST API (JSON)
                      │ SSE (流式AI对话)
┌─────────────────────▼────────────────────────────┐
│               Backend (SpringBoot 3.3)            │
│  Controller → Service → Mapper (MyBatis-Plus)     │
│  Security: JWT + BCrypt                          │
│  Cache: Redis (Lettuce)                          │
│  AI: OpenAI-Compatible API (SSE Stream)          │
│               Port: 8080                          │
└─────────────────────┬────────────────────────────┘
                      │
┌─────────────────────▼────────────────────────────┐
│                  Data Layer                        │
│  MySQL 8.0（达梦 DM8 可选）│  Redis 7.x            │
└──────────────────────────────────────────────────┘
```

### 3.2 用户认证模块

- **技术栈**：JWT（jjwt 0.11.5）+ BCrypt（spring-security-crypto）
- **流程**：POST `/api/auth/login` → 验证BCrypt密码 → 生成JWT令牌(含角色信息，120分钟有效期) → 前端存入 localStorage
- **安全特性**：密码 BCrypt 加密存储；JWT 令牌过期自动跳转登录；前后端分离CORS白名单

### 3.3 学生管理模块

- **接口**：GET/POST/PUT/DELETE `/api/students`
- **功能**：分页查询、关键词搜索（姓名/学号）、性别/专业筛选、排序、批量删除
- **缓存**：Redis 缓存学生列表（10分钟TTL），写操作自动清除缓存

### 3.4 班级管理模块

- **接口**：GET/POST/PUT/DELETE `/api/classes`
- **功能**：班级 CRUD、辅导员分配、年级/专业筛选、学生人数统计

### 3.5 课程管理模块（新增）

- **接口**：GET/POST/PUT/DELETE `/api/courses`
- **功能**：课程信息全生命周期管理、学分/学期/容量设置、教师分配
- **缓存**：Redis 缓存课程列表（15分钟TTL）

### 3.6 选课管理模块（新增）

- **接口**：GET/POST/PUT/DELETE `/api/enrollments`
- **功能**：学生选课（防重复）、退课、成绩录入、按学生/课程维度查询
- **约束**：UNIQUE(student_id, course_id) 防止重复选课

### 3.7 AI 智能助手模块（新增）🌟

- **接口**：POST `/api/ai/chat`（SSE 流式输出）、GET `/api/ai/history`、DELETE `/api/ai/history`
- **核心特性**：
  1. **流式对话**（SSE）：基于 Server-Sent Events 实现打字机效果输出
  2. **上下文记忆**：携带历史对话上下文，支持多轮连续对话
  3. **对话历史持久化**：MySQL 存储所有对话记录，支持回溯
  4. **服务降级**：OpenAI API 不可用时，自动切换为本地规则引擎响应
  5. **OpenAI 兼容**：支持任意 OpenAI 协议兼容接口（包括国产大模型如 DeepSeek、通义千问等）
- **前端 UI**：专用聊天界面，含快捷问题、加载历史、清空历史功能
- **提示词**：System prompt 定制为「学生管理系统AI助手」角色

## 四、SQL 语句优化

### 4.1 索引优化策略

系统中所有高频查询字段均建立了索引，具体策略如下：

| 表 | 索引 | 类型 | 优化场景 |
|---|------|------|---------|
| student | `uk_student_no` | UNIQUE | 学号精确查询（高频） |
| student | `idx_student_name` | NORMAL | 姓名模糊搜索 |
| student | `idx_student_major` | NORMAL | 按专业筛选 |
| student | `idx_student_class_id` | NORMAL | 按班级查询学生列表 |
| student | `idx_student_name_major` | **复合索引** | 姓名+专业联合查询（覆盖索引） |
| course | `idx_course_name` | NORMAL | 课程名称搜索 |
| course | `idx_course_teacher` | NORMAL | 教师课程查询 |
| course | `idx_course_semester` | NORMAL | 学期课程列表 |
| enrollment | `uk_student_course` | UNIQUE | 防重复选课+快速查询 |
| enrollment | `idx_enrollment_student` | NORMAL | 学生选课列表（最左前缀） |
| enrollment | `idx_enrollment_course` | NORMAL | 课程选课人数统计 |
| ai_conversation | `idx_conv_user_created` | **复合索引** | 用户对话历史按时间排序 |

### 4.2 索引设计原则

1. **最左前缀原则**：复合索引 `idx_student_name_major(name, major)` 支持仅按 `name` 查询
2. **覆盖索引**：`idx_enrollment_score` 支持成绩统计查询直接从索引获取数据
3. **避免过度索引**：仅对高频查询字段建索引，写入操作不会因过多索引而变慢
4. **EXPLAIN 验证**：所有查询均使用 `EXPLAIN` 分析执行计划，确认 type 为 `ref` 或 `const`

### 4.3 查询优化示例

```sql
-- 优化前：全表扫描
SELECT * FROM student WHERE name LIKE '%张%' AND major = '计算机科学与技术';

-- 优化后：使用复合索引 idx_student_name_major
EXPLAIN SELECT * FROM student WHERE name LIKE '张%' AND major = '计算机科学与技术';
-- type: range, key: idx_student_name_major, rows: <10

-- 选课查询优化：使用覆盖索引
EXPLAIN SELECT score FROM enrollment WHERE student_id = 1;
-- type: ref, key: idx_enrollment_student, Extra: Using index
```

### 4.4 MyBatis-Plus 分页优化

所有列表查询均使用 MyBatis-Plus 分页插件（`PaginationInnerInterceptor`），自动拦截 SQL 并添加 `LIMIT` 子句，避免大数据量查询导致内存溢出。

## 七、部署说明

### 7.1 环境要求

- JDK 17+
- Node.js 18+
- MySQL 8.0+
- Redis 7.x（可选，用于缓存）

### 7.2 后端启动

```bash
cd backend
# 修改 application.yml 中的数据库连接和 AI API Key
mvn spring-boot:run
# 访问 http://localhost:8080/swagger-ui.html 查看接口文档
```

### 7.3 前端启动

```bash
cd frontend
pnpm install
pnpm dev
# 访问 http://localhost:5173
# 默认管理员: admin / admin123
```

### 7.4 AI 功能配置

在 `application.yml` 中配置：
```yaml
ai:
  openai:
    api-key: ${OPENAI_API_KEY:sk-your-api-key}
    base-url: https://api.openai.com    # 可替换为国产模型地址
    model: gpt-3.5-turbo
```

即使不配置 API Key，AI 助手也能以**降级模式**运行，提供基于规则引擎的智能响应。

### 7.5 国产数据库扩展（达梦 DM8，可选）

本项目已预置**达梦 DM8** 国产数据库支持，需要时通过 Spring Profile 一键切换：

```bash
# 切换为达梦 DM8（国产数据库）
mvn spring-boot:run -Dspring-boot.run.profiles=dm8
```

**前置准备**：
1. 安装达梦 DM8 数据库（开发版免费，[eco.dameng.com](https://eco.dameng.com/download/)）
2. 使用 `schema-dm8.sql` 初始化数据库（位于 `src/main/resources/`）
3. 修改 `application-dm8.yml` 中的数据库连接信息

**核心配置**（`application-dm8.yml`）：
```yaml
spring:
  datasource:
    driver-class-name: dm.jdbc.driver.DmDriver
    url: jdbc:dm://127.0.0.1:5236/student_system?charSet=utf8&compatibleMode=oracle
    username: SYSDBA
    password: SYSDBA
```

**MySQL → DM8 SQL 语法适配**（已完成的改造）：
| MySQL | DM8（Oracle 兼容） |
|-------|-------------------|
| `DROP TABLE IF EXISTS` | `BEGIN ... EXCEPTION WHEN OTHERS THEN NULL; END;` |
| `AUTO_INCREMENT` | `IDENTITY` |
| `VARCHAR(n)` | `VARCHAR2(n)` |
| `TEXT` | `CLOB` |
| `DECIMAL(p,s)` | `NUMBER(p,s)` |
| `DATETIME` | `TIMESTAMP` |
| `ENGINE=InnoDB` | 无需指定 |
